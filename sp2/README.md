# Distributed Parallel Chat System
**Course:** Methods and Technologies of Parallel Programming  
**Project:** №2 — Client-Server Chat System

---

## Overview

A real-time chat system built with a client-server architecture where the server handles multiple concurrent clients using one thread per connection. Supports broadcast messages, private messages, group chats, file transfer, markdown formatting, offline message delivery, and automated load testing with performance graphs.

## Running the Project

### Mode 1 — Demo (1 terminal, fully automatic)

Starts the server, connects 10 bots for 10 seconds, then generates performance graphs automatically.

```bash
python main.py demo
```

Output: PNG graphs appear in `results/`

---

### Mode 2 — Manual chat (2 terminals)

**Terminal 1 — start the server:**
```bash
python main.py server
```

**Terminal 2 — connect a client:**
```bash
python main.py client
```

You can open as many client terminals as you want. Each runs independently.

---

### Mode 3 — Benchmark (2 terminals)

**Terminal 1 — start the server first:**
```bash
python main.py server
```

**Terminal 2 — run the load test:**
```bash
python main.py bench --clients 10 --duration 20
```

After the benchmark finishes, `results/benchmark_raw.json` is created.

---

### Mode 4 — Generate graphs (1 terminal)

Run after benchmark to produce PNG charts:

```bash
python generate_graphs.py
```

Or via the launcher:

```bash
python main.py graphs
```

## Launcher Arguments

```
python main.py <mode> [options]

Modes:
  server    Start the chat server
  client    Connect as an interactive client
  bench     Run automated load test (server must be running)
  graphs    Generate PNG graphs from benchmark/stats results
  demo      Run everything automatically (server + bots + graphs)

Options:
  --host      Server host to connect to (default: 127.0.0.1)
  --port      Port number (default: 9090)
  --clients   Number of bot clients for bench (default: 10)
  --duration  Benchmark duration in seconds (default: 20)
```

## Client Commands

Once connected, type these in the client terminal:

| Command | Description |
|---|---|
| `<text>` | Broadcast message to all users |
| `/pm <user> <text>` | Private message to a specific user |
| `/join <group>` | Join a group |
| `/leave <group>` | Leave a group |
| `/group <group> <text>` | Send message to a group |
| `/file <user> <path>` | Send a file to a user (max 50 MB) |
| `/gfile <group> <path>` | Send a file to a group |
| `/me <text>` | Action message (e.g. `/me waves`) |
| `/md <text>` | Send message with Markdown formatting |
| `/list` | Show online users |
| `/groups` | Show groups and their members |
| `/help` | Show help |
| `/quit` | Disconnect |

Downloaded files are saved to `client/downloads/`.

## Architecture

- **Concurrency model:** one `threading.Thread` per client connection
- **Shared state protection:** single `threading.Lock` guards `clients`, `groups`, `offline` dictionaries
- **Message framing:** 4-byte big-endian length prefix + JSON payload
- **Offline delivery:** messages to disconnected users are queued and delivered on reconnect
- **Max file size:** 50 MB (base64-encoded in JSON frame)
